Introduction

HTML (HyperText Markup Language) is the core language used to structure content on the web. It defines how elements like text, images, and links appear on a webpage. While HTML provides structure, CSS is used for styling, and JavaScript adds interactivity.

"Hypertext" refers to the links that connect web pages, making navigation possible. HTML uses "markup" to organize content with elements such as <head>, <title>, <body>, <p>, <img>, and many more, which the browser interprets to display the content.

HTML tags, written inside < >, are case-insensitive but are generally written in lowercase for consistency.

What You Should Already Know

Before diving into HTML, it's helpful to have some basic knowledge of the following:

  • How the Internet Works: Understanding the basics of how web browsers and servers communicate will make it easier to grasp HTML's role.
  • Basic Text Editing Skills: Knowing how to create and save HTML files using a text editor is essential.
  • Navigating a Web Browser: Being familiar with how to open and browse web pages will help you test and see your HTML in action.

HTML Elements

In HTML, an element consists of:

  1. Opening Tag: This marks the beginning of the element, such as <h1>.
  2. Content: The information or text inside the element, like "Welcome to My Page."
  3. Closing Tag: This marks the end of the element, such as </h1>.
            
    <h1>Welcome to My Page</h1>

In this example:

  • <h1> is the opening tag.
  • "Welcome to My Page" is the content.
  • </h1> is the closing tag.

The structure follows the pattern: <tag> content </tag>.

Some elements may not require a closing tag (like the <img> element), but most elements follow this basic structure.

HTML Attributes

HTML attributes provide additional information about an element and are placed within the opening tag. They are used to configure or modify the behavior or appearance of HTML elements. Attributes consist of a name and a value, and they help define properties such as the source of an image or the link destination of a hyperlink.

            
    <a href="https://www.example.com">Visit Example</a>

In this example, the href attribute specifies the URL (https://www.example.com) the link points to.

HTML Core Structure

The core structure of an HTML document consists of several key elements that define the content and structure of a webpage. At the top, we have the <!DOCTYPE html> declaration, which defines the document type as HTML5. The root element is <html>, which contains the entire document, and is typically assigned a "lang" attribute to specify the language of the content. Inside the <html> element, the <head> section contains metadata, links to stylesheets, and scripts, including the <meta> tags for character encoding and viewport settings. The <body> section holds the visible content of the page, such as headings, paragraphs, images, links, and other elements that users interact with. This basic structure provides the foundation for any webpage, and additional sections can be added depending on content's needs, like navigation, forms, and multimedia.

Hello World

HTML (Hypertext Markup Language) serves as the core structure of web pages. One of the fundamental elements in HTML is headings. With headings, we can easily display "Hello World!" on a webpage like this:

            
    <!DOCTYPE html>
    <html lang="en">
      <head>
        <title>Hello World Page</title>
      </head>
      <body>
        <h1>Hello World!</h1>
      </body>
    </html>

Notice that we use indentation using "Tab" on the keyboard to tell that an element is within another element. This section <h1>Hello World</h1> guarantees that when you save the code as a text file named index.html and open it in your browser, the text "Hello World!" will be displayed.

Heading Element

Headings in HTML range from <h1> to <h6>, and they are used to define the structure and importance of content on a webpage.

  • <h1>: Represents the most important heading and is typically used for the main title or key content of the page.
  • <h2>: Used for subheadings, representing sections under the main title.
  • <h3>: Represents subsections under <h2>, and so on for deeper content hierarchy.
  • <h4> to <h6>: These are used for further subdivisions and lesser importance, with <h6> being the least important.

We can write it like this:

            
    <h1>I'm the Most Important Thing Here!</h1>
    <h2>Second in Line, Still Pretty Important</h2>
    <h3>Not As Big, But Still Noteworthy</h3>
    <h4>Getting a Little Less Important</h4>
    <h5>Kind of a Side Note</h5>
    <h6>Just a Tiny Detail</h6>

Headings provide a semantic structure to the content, helping with both readability and accessibility. It's essential to use them in order, starting with <h1> and proceeding through to <h6>.

Paragraph Element

The <p> HTML element represents a paragraph of text, typically displayed as a block separated by vertical space or indentation from adjacent blocks. Browsers automatically add space before and after paragraphs to enhance readability.

            
    <p>This is the first paragraph of text.</p>
    <p>This is the second paragraph.</p>

To adjust the spacing or appearance of paragraphs, it's recommended to use CSS. For instance, to increase the space between paragraphs and improve line spacing.

It's important to note that using empty <p> elements to create space between paragraphs is discouraged, as this can cause issues for users relying on assistive technologies. Instead, CSS should be used to manage spacing and layout.

Anchor Element

The <a> (anchor) element in HTML is used to create hyperlinks, allowing users to navigate to other web pages or locations within the same page. It requires the href attribute to specify the destination.

            
    <a href="https://www.google.com/">Google</a>

This creates a link that takes the user to google.com when clicked.

To open a link in a new tab, use the target="_blank" attribute: 

            
    <a href="https://www.google.com/" target="_blank">Google</a>

For linking to a section on the same page:

            
    <a href="#section1">Go to Section 1</a>
    <p id="section1">This is Section 1.</p>

In this case we also use an id attribute, which we'll touch on later.

Title Element

The <title> element defines the document's title, displayed in the browser's title bar or tab. It should be concise, descriptive, and unique for each page, aiding in search engine optimization (SEO) and user navigation. The <title> element must be placed within the <head> section of the HTML document.

            
    <head>
      <title>Introduction to HTML</title>
    </head>

Image Element

The <img> element embeds images in an HTML page. It requires the src attribute for the image URL and the alt attribute for accessibility.

            
    <img src="https://example.com/image.jpg" alt="Description of image">

To control the size, use the width and height attributes:

            
    <img src="image.jpg" alt="Example image" width="200" height="150">

Use CSS for better styling and responsiveness.

List Element

The <ul>, <ol>, <dl> and <li> elements create lists in HTML. The <li> is used as the content of the list.

Unordered list (<ul>) displays items with bullet points:

            
    <ul>
      <li>Apple</li>
      <li>Banana</li>
      <li>Orange</li>
    </ul>

Ordered list (<ol>) displays items with numbers:

            
    <ol>
      <li>First Step</li>
      <li>Second Step</li>
      <li>Third Step</li>
    </ol>

Description list (<dl>) defines terms and descriptions using <dt> and <dd> respectively:

            
    <dl>
      <dt>HTML</dt>
      <dd>A markup language for web pages.</dd>
    </dl>

Lists help organize content clearly.

Div Element

The <div> (division) element is a generic container for grouping and styling content in HTML. It has no inherent semantic meaning and doesn't affect content or layout until styled with CSS or manipulated with JavaScript.

            
    <div class="note">
      <p>This is a note.</p>
    </div>

The <div> element is commonly used with CSS and JavaScript for layout and interactivity.

Span Element

The <span> element is an inline container used for styling a part of text or grouping inline elements without adding extra meaning. It does not affect the layout on its own.

            
    <p>This is a <span class="highlight">highlighted</span> word.</p>

We can then style it using CSS.

The <span> element is useful for applying styles or scripting to small text portions without affecting the document's structure.

Table Element

The <table> element is used to create tables in HTML. It organizes data into rows and columns using child elements like <tr>, <th>, and <td>.

            
    <table>
      <tr>
        <th>Name</th>
        <th>Age</th>
      </tr>
      <tr>
        <td>Alice</td>
        <td>25</td>
      </tr>
      <tr>
        <td>Bob</td>
        <td>30</td>
      </tr>
    </table>

Explanation:

  • <tr> defines a row.
  • <td> defines a header cell.
  • <td> defines a standard data cell.

Tables help structure tabular data for better readability and accessibility.

Form Element

The <form> element in HTML is used to collect user input and submit it to a server for processing. It acts as a container for various input elements like text fields, checkboxes, radio buttons, and submit buttons.

            
    <form action="/submit-form" method="post">
      <label for="username">Username:</label>
      <input type="text" id="username" name="username" required>

      <label for="password">Password:</label>
      <input type="password" id="password" name="password" required>

      <button type="submit">Login</button>
    </form>

In this example:

  • The action attribute specifies that the form data will be submitted to /submit-form.
  • The method can be set to GET or POST. In this case, the method is set to POST, meaning the data will be sent in the body of the HTTP request.
  • Each <label> is associated with an <input> field through the for and id attributes, enhancing accessibility.

Section Element

The <section> element groups related content in a webpage. It helps organize content into meaningful sections. The main purposes include:

  • Used for thematically related content.
  • Often contains a heading (<h1>-<h6>).
  • Improves document structure and accessibility.
            
    <section>
      <h2>About Us</h2>
      <p>We provide web development tutorials.</p>
    </section>

Article Element

The <article> element represents self-contained content that can be independently distributed, like blog posts or news articles. It is used for a standalone content and often includes a heading and text. <article> helps with structuring and reusability.

            
    <article>
      <h2>Latest News</h2>
      <p>New web development trends are emerging this year.</p>
    </article>

Id Attribute

The id attribute assigns a unique identifier to an element, used for linking (#id in URLs), JavaScript (document.getElementById()), and CSS styling (#id). It must be unique within a document and should avoid whitespace.

            
    <p id="intro">Welcome!</p>

Note:

  • id selectors in CSS have high specificity, making styles harder to override.
  • For flexibility, prefer class selectors in styling and data-* attributes in JavaScript when possible.

Class Attribute

The class attribute in HTML assigns one or more class names to an element, facilitating CSS styling and JavaScript manipulation. Multiple class names should be separated by spaces.

            
    <p class="note editorial">This is a note.</p>

You can target these classes in CSS to apply styling.

Best practices for class

  • Semantic Naming: Use class names that reflect the purpose or meaning of the element, rather than its appearance. For instance, prefer .warning over .red-text.
  • Valid Identifiers: While class names can include any Unicode characters (except ASCII whitespace), it's advisable to use valid CSS identifiers to avoid the need for escaping characters in CSS selectors.

Href Attribute

The href (Hypertext Reference) attribute defines the URL or resource a link points to. It is used in:

<a> (Anchor) element for creating hyperlinks.

            
  <a href="https://example.com">Visit Example</a>

<link> (Link) element for linking external sources

            
  <link rel="stylesheet" href="styles.css">

Src Attribute

The src (source) attribute specifies the location of an external resource for various HTML elements:

<img> for displaying images.

 
            
  <img src="image.jpg" alt="Description">

<script> for linking Javascript external files.

 
            
  <script src="script.js"></script>

<iframe> for embedding another web page.

 
            
  <iframe src="https://example.com"></iframe>

<audio> and <video> for embedding media files

 
            
  <video src="video.mp4" controls></video>

Key points of using src:

  • Accepts absolute (https://example.com/file.js) and relative (file.js) URLs.
  • Ensures proper loading of images, scripts, and media content.

Alt Attribute

The alt (alternative text) attribute provides a text description for images in the <img> element. The main purposes are to improve accessibility for screen readers, to displays text if the image fails to load, and to help with SEO by providing context to search engines.

            
    <img src="image.jpg" alt="A scenic mountain view">

It's best to keep alt text descriptive but concise. Use an empty alt="" for purely decorative images.

Title Attribute

The title attribute provides additional information about an element, shown as a tooltip on hover.

Key points:

  • Provides extra details but shouldn't be the only source of important information.
  • Not always accessible on touchscreens or assistive technologies.
  • Best used for non-essential hints or descriptions.
            
    <a href="https://example.com" title="Visit Example website">Example</a>

For better accessibility, prefer visible text or ARIA attributes over title.

Style Attribute

The style attribute is a global attribute that contains CSS styling declarations to be applied to an element. However, it is recommended to define styles in separate files or within a <style> element in the document's <head> for better maintainability.

            
    <p style="color: blue; font-size: 14px;">This is a styled paragraph.</p>

In this example, the paragraph text will appear in blue with a font size of 14 pixels.

Couple things to consider

  • Specificity: Inline styles have higher specificity compared to styles defined in <style> tags or external stylesheets.
  • Maintainability: Using inline styles can make your HTML code harder to maintain. It's often better to define styles in a <style> tag within the <head> section or in an external CSS file.

Target Attribute

The target attribute in HTML specifies where to display the linked resource when a hyperlink is clicked. It's commonly used with the <a> (Anchor) element and can also be applied to <area>, <form>, and <base> elements. Most used values for target:

  • _self (default): Opens in the same tab.
  • _blank: Opens in a new tab/window.
  • _parent: Opens in the parent frame.
  • _top: Opens in the full window.
            
    <a href="https://example.com" target="_blank" rel="noopener">Visit Example</a>

For security, add rel="noopener" with _blank to prevent access to window.opener.

Placeholder Attribute

The placeholder attribute in HTML specifies a short hint that describes the expected value of an input field or textarea. This hint is displayed inside the input field before the user enters a value.

            
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" placeholder="Enter your username">

In this example, the input field will display "Enter your username" as a placeholder text until the user types in the field.

Couple things to consider

  • Not a Label Substitute: Placeholders disappear when typing, so always use <label> for accessibility.
  • Accessibility: Relying solely on placeholders for instructions can be problematic for users with cognitive or visual impairments. Always use <label> elements to ensure form fields are accessible.

Value Attribute

The value attribute in HTML sets the initial value of form elements such as <input>, <button>, and <option>. Example of usages:

Text Input Field

            
  <input type="text" value="Default Text">

Displays an input field pre-filled with "Default Text".

Radio Button

            
  <input type="radio" name="choice" value="option1">

Assigns "option1" as the value sent when this radio button is selected.

Button Element

            
  <button type="submit" value="SubmitForm">Submit</button>

Associates "SubmitForm" as the value when this button is clicked.

Note: The value attribute is crucial for form data submission, as it determines the data sent to the server upon form submission.

Name Attribute

The name attribute assigns an identifier to form controls like <input>, <select>, and <textarea>, facilitating data organization upon form submission. Each control's name and value are sent as a pair to the server.

            
    <form action="/submit" method="post">
      <label for="email">Email:</label>
      <input type="email" id="email" name="user_email">
      <button type="submit">Submit</button>
    </form>

In this example, the input field's name is user_email. Upon submission, the server receives the input's data under this identifier.

Note: Proper use of the name attribute ensures accurate data handling during form processing.

Comment

In HTML, comments are used to insert notes or prevent code from being executed. They begin with <!--< and end with -->.

            
    <!-- This is a comment -->
    <p>This paragraph is visible.</p>
    <!-- <p>This paragraph is hidden.</p> -->

Several points to consider:

  • Placement: Comments can be placed before or after the <!DOCTYPE> declaration and the <html> element.
  • Restrictions: Avoid placing comments within tags, as they are only permitted as content. Additionally, comments cannot contain the string -- or start with > or ->.
  • Non-Nesting: Comments cannot be nested. The first occurrence of --> closes the comment.

Proper use of comments enhances code readability and aids in documentation.
